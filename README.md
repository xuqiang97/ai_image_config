# AI生图管理员配置页面需求

## 1. 说明

`index.html` 是本需求的页面原型，前端实现以原型和本文档为准。线上原型地址：[https://xuqiang97.github.io/ai_image_config/](https://xuqiang97.github.io/ai_image_config/)。

本需求对应的后端接口已提前上线。此前因暂无前端资源，产品侧先通过 Postman 使用接口；本期主要补齐前端页面，不新增后端接口、不重做后端设计。若前端联调发现接口字段、权限、校验或返回行为与页面需求不一致，联系后端联调支持。

## 2. 范围

需要实现两个独立页面或路由：

| 页面 | 用途 | 接口区分 |
| --- | --- | --- |
| 美工管理员配置 | 管理美工场景下的 AI 生图配置 | 按已上线接口约定匹配“美工”配置类型 |
| 运营管理员配置 | 管理运营场景下的 AI 生图配置 | 按已上线接口约定匹配“运营”配置类型 |

页面顶部统一展示：

> 由指定管理员统一配置，其他人请勿动谢谢，排序默认按更新时间倒排

范围约束：

- 两个页面使用独立菜单/路由；页面内不展示底层表名或接口技术标识。
- 两个页面交互和字段规则一致，但数据范围、接口入参或配置类型需严格区分。
- 切换页面时清空筛选、分页和表单状态，并回到第一页。
- 非本期范围：新增后端接口、数据库调整、审批流、批量导入导出、复杂审计报表。

## 3. 权限与接口边界

- 菜单/路由权限控制用户是否能进入页面。
- 按钮权限如后台已有细粒度控制，应覆盖新增、编辑、删除、批量更新。
- 接口权限以后端已上线鉴权为准；前端隐藏入口或按钮不能替代后端鉴权。
- 无权限时按后台规范展示无权限提示或跳转无权限页面。
- 接口成功后按操作刷新列表；接口失败时优先展示接口返回文案。
- 查询、提交、删除、批量更新时应有加载态或按钮禁用态，避免重复提交。
- 列表无数据展示空状态，不应误报为接口失败。

## 4. 列表页

筛选条件：

| 字段 | 规则 |
| --- | --- |
| 用户ID | 精确查询 |
| 用户名 | 完整用户名查询 |
| 生图模型 | 下拉选择 |
| appKey | 精确查询 |
| 启用状态 | 全部、已启用、已停用 |

列表字段：

| 字段 | 说明 |
| --- | --- |
| ID | 配置记录 ID |
| 用户ID | 用户标识 |
| 用户名 | 用户完整名称 |
| 生图模型 | 模型名称，按后端模型编码映射展示 |
| appKey | 生图调用 appKey |
| quota | JSON 对象，列表截断展示，详情完整展示 |
| 状态 | 已启用 / 已停用 |
| 更新时间 | 默认按更新时间倒排；相同更新时间按 ID 倒排 |
| 操作 | 查看、编辑、删除 |

分页与重置：

- 默认 20 条/页，支持 20/50/100 条切换。
- 支持上一页、下一页和页码跳转。
- 点击查询后从第一页展示结果。
- 点击重置后清空筛选条件，回到第一页，并重新拉取当前页面数据。

生图模型选项以已上线接口或后台枚举为准；原型包含：

| 编码 | 展示名称 |
| --- | --- |
| 1 | LinkFox（1） |
| 2 | 自研（2） |
| 3 | 阿里（3） |
| 4 | 豆包（4） |

未知编码展示为“未知生图模型（编码）”，便于排查数据。

## 5. 配置操作

| 操作 | 要求 |
| --- | --- |
| 新增 | 打开右侧抽屉；用户ID、生图模型、用户名、appKey、quota 必填；启用状态默认已启用 |
| 查看 | 打开右侧抽屉；展示基础信息、创建时间、更新时间和完整 quota JSON；支持从查看进入编辑 |
| 编辑 | 用户ID、生图模型、用户名只读；appKey、quota、启用状态可编辑 |
| 删除 | 需二次确认，提示用户和用户ID；成功后刷新列表 |
| 批量更新 | 按 appKey 命中，可选生图模型缩小范围；只对命中记录的 quota 做整包覆盖 |

新增校验：

- 用户ID必须为整数。
- appKey、用户名、quota 不能为空。
- 同一页面范围内，同一用户ID + 同一生图模型只允许一条配置。
- 重复时提示“该用户下该生图模型配置已存在”。
- 新增成功后关闭抽屉、刷新列表并回到第一页。

批量更新规则：

- 生图模型不选时，命中该 appKey 下全部生图模型配置。
- quota 以完整对象整包覆盖，不做字段级合并。
- 抽屉顶部警示文案即可满足风险提示，不强制增加二次确认弹窗；如项目已有批量操作规范，按项目规范补充。
- 未命中记录时提示“未命中任何记录”。

## 6. quota 规则

quota 支持结构化填写和 JSON 直填，两种方式提交前使用同一套校验。

| 规则项 | 要求 |
| --- | --- |
| 类型 | 必须为 JSON 对象 |
| 空对象 | 不允许 |
| key | 仅允许字母、数字、下划线，且必须以字母或下划线开头 |
| key 重复 | 不允许 |
| value | 必须为 0-20 的整数 |

示例：

```json
{
  "total": 20,
  "scene": 3,
  "closeUp": 2
}
```

本期前端先按 `0-20` 做提交前校验。若接口规则更细或与本文档冲突，展示接口错误并联系后端和产品确认最终口径。

## 7. 交互要求

- 抽屉和删除弹窗支持关闭按钮与 Esc 关闭。
- 抽屉打开时聚焦到关闭按钮或首个可操作元素，关闭后尽量恢复到触发入口。
- 筛选标签展示用户输入时必须转义。
- 列表中 quota 可截断展示，详情中必须完整展示。
- 美工管理员配置和运营管理员配置统一使用蓝色主题。

## 8. 验收重点

| 模块 | 验收点 |
| --- | --- |
| 页面入口 | 两个独立页面/路由可访问，命名正确，接口请求能区分美工/运营 |
| 权限 | 页面、按钮、接口权限边界清晰，无权限状态处理正确 |
| 查询列表 | 筛选、重置、分页、排序、空状态符合要求 |
| 配置操作 | 新增、查看、编辑、删除、批量更新流程完整 |
| quota | 结构化填写和 JSON 直填规则一致，错误提示清晰 |
| 接口状态 | 加载、成功、失败、无权限、防重复提交均有处理 |
| 交互 | 抽屉/弹窗关闭、Esc、焦点恢复、主题色符合原型 |

## 9. 开放问题

- 指定管理员名单和权限配置方式以前端所在后台权限体系为准。
- 生图模型枚举是否扩展，以后端接口或统一枚举源为准。
- 如后续确认不同生图模型的 quota value 范围不同，需补充模型维度校验规则；本期统一按 `0-20` 执行。
