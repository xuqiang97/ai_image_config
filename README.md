# AI生图管理员配置页面需求

## 1. 文档说明

本文档用于指导前端实现 `AI生图管理员配置原型` 对应的页面需求。当前仓库中的 `index.html` 是页面交互原型，前端实现时以该原型和本文档为主要依据。

本需求对应的后端接口已在前期单独上线。当时因暂无前端资源，后端接口先行交付，产品侧已通过 Postman 临时使用。因此本次需求以纯前端页面实现为主，不要求新增后端接口或重做后端设计。若前端实现或联调过程中发现接口行为、字段、权限或数据返回与页面需求不一致，前端同学可联系后端进行联调支持。

## 2. 背景与目标

AI 生图能力需要按用户、appKey 和生图模型维护管理员配置，包括额度 `quota`、启用状态等信息。当前配置依赖后端接口和 Postman 操作，使用成本较高，也不利于指定管理员日常维护。

本需求目标是在前端后台中提供可视化配置页面，让指定管理员可以完成查询、新增、查看、编辑、删除和按 appKey 批量更新 quota 等操作。

## 3. 页面范围

需要实现两个独立页面或路由：

| 页面 | 用途 | 实现说明 |
| --- | --- | --- |
| 美工管理员配置 | 管理美工场景下的 AI 生图配置 | 独立菜单/路由；请求时需按已上线接口约定传入或匹配“美工”配置类型 |
| 运营管理员配置 | 管理运营场景下的 AI 生图配置 | 独立菜单/路由；请求时需按已上线接口约定传入或匹配“运营”配置类型 |

两个页面的交互和字段规则保持一致，仅数据范围和接口入参/配置类型不同。页面顶部统一展示提示文案：

> 由指定管理员统一配置，其他人请勿动谢谢，排序默认按更新时间倒排

页面区分要求：

- 两个页面应有独立菜单入口和独立路由，避免用户误以为只是同一页面的筛选切换。
- 两个页面请求接口时必须明确区分配置类型。具体使用接口路径、业务类型字段或其他标识，以已上线接口契约为准。
- 前端页面不展示底层表名或接口技术标识，只展示业务名称“美工管理员配置”“运营管理员配置”。
- 两个页面的筛选状态、分页状态和表单状态互不复用；切换页面时清空筛选条件并回到第一页。

## 4. 非本期范围

- 不新增后端接口。
- 不设计或调整数据库表结构。
- 不新增审批流、操作工单流。
- 不做批量导入、批量导出。
- 不做复杂审计报表。若已有后端操作日志能力，可按现有后台规范接入。

## 5. 权限要求

页面入口应仅对指定管理员开放。若当前后台已有菜单/路由权限体系，前端按既有方式接入。

权限重点：

- 菜单/路由权限：控制用户是否能看到并进入“美工管理员配置”“运营管理员配置”页面。
- 按钮权限：如后台已有细粒度按钮权限，应分别控制新增、编辑、删除、批量更新等配置变更操作。
- 接口权限：以后端已上线接口鉴权为准，前端隐藏入口或按钮不能替代后端鉴权。
- 非指定管理员不应看到或进入配置页面。
- 新增、编辑、删除、批量更新属于配置变更操作，前端需避免无权限用户触发操作。
- 后端接口权限以已上线实现为准；若联调发现前端权限和接口权限不一致，需联系后端确认处理。
- 用户无权限进入页面或调用接口返回无权限时，应按后台统一规范展示无权限提示或跳转无权限页面。

## 6. 列表页需求

### 6.1 筛选条件

列表页支持以下筛选：

| 字段 | 规则 |
| --- | --- |
| 用户ID | 精确查询 |
| 用户名 | 完整用户名查询 |
| 生图模型 | 下拉选择 |
| appKey | 精确查询 |
| 启用状态 | 全部、已启用、已停用 |

切换“美工管理员配置”和“运营管理员配置”页面时，应清空上一页面的筛选条件，避免把一个页面的筛选状态带到另一个页面。

### 6.2 列表字段

列表展示以下字段：

| 字段 | 说明 |
| --- | --- |
| ID | 配置记录 ID |
| 用户ID | 用户标识 |
| 用户名 | 用户完整名称 |
| 生图模型 | 模型名称，按后端模型编码映射展示 |
| appKey | 生图调用 appKey |
| quota | JSON 对象，列表中可折叠/截断展示，详情中完整展示 |
| 状态 | 已启用 / 已停用 |
| 更新时间 | 记录最后更新时间 |
| 操作 | 查看、编辑、删除 |

默认排序按更新时间倒排；如更新时间相同，可按 ID 倒排。

### 6.3 生图模型选项

原型中的生图模型选项如下，最终以已上线接口或后台枚举为准：

| 编码 | 展示名称 |
| --- | --- |
| 1 | LinkFox（1） |
| 2 | 自研（2） |
| 3 | 阿里（3） |
| 4 | 豆包（4） |

如接口返回未知编码，前端可展示为“未知生图模型（编码）”，便于排查数据问题。

### 6.4 分页与重置

列表需支持分页能力：

- 默认每页 20 条。
- 支持切换为 20 条/页、50 条/页、100 条/页。
- 支持上一页、下一页和页码跳转。
- 点击“查询”后，从第一页展示查询结果。
- 点击“重置”后，清空所有筛选条件，回到第一页，并重新拉取当前页面范围的数据。
- 切换“美工管理员配置”和“运营管理员配置”时，清空筛选条件和分页状态，回到第一页。

## 7. 新增配置

点击“新增配置”后打开右侧抽屉。

### 7.1 字段

| 字段 | 是否必填 | 规则 |
| --- | --- | --- |
| 用户ID | 是 | 必须为整数 |
| 生图模型 | 是 | 从下拉选项中选择 |
| 用户名 | 是 | 必须填写完整用户名 |
| appKey | 是 | 不能为空 |
| quota | 是 | 见 quota 规则 |
| 启用状态 | 是 | 默认已启用 |

### 7.2 新增校验

- 用户ID、用户名、生图模型、appKey、quota 均不能为空。
- 同一页面范围内，同一用户ID + 同一生图模型只允许存在一条配置。
- 若已存在，提示“该用户下该生图模型配置已存在”。
- 新增成功后关闭抽屉，刷新列表，并回到第一页。

## 8. 查看配置

点击列表“查看”后打开右侧抽屉，展示完整配置详情。

详情字段包括：

- ID
- 用户ID
- 用户名
- 生图模型
- appKey
- 启用状态
- 创建时间
- 更新时间
- 完整 quota JSON

查看抽屉支持直接点击“编辑”进入编辑状态。

## 9. 编辑配置

点击列表“编辑”或详情抽屉中的“编辑”后打开编辑抽屉。

### 9.1 可编辑字段

| 字段 | 编辑规则 |
| --- | --- |
| 用户ID | 只读 |
| 生图模型 | 只读 |
| 用户名 | 只读 |
| appKey | 可编辑，不能为空 |
| quota | 可编辑，见 quota 规则 |
| 启用状态 | 可编辑 |

编辑成功后关闭抽屉并刷新列表。

## 10. 删除配置

点击列表“删除”后弹出确认弹窗。

弹窗需明确提示将删除的用户和用户ID，例如：

> 确认删除用户 XXX（用户ID：123）这条配置吗？删除后不可恢复。

用户确认后调用已上线删除接口，成功后关闭弹窗并刷新列表。

## 11. 按 appKey 批量更新

点击“按 appKey 批量更新”后打开右侧抽屉。

### 11.1 命中条件

| 字段 | 是否必填 | 规则 |
| --- | --- | --- |
| appKey | 是 | 精确命中 |
| 生图模型 | 否 | 不选时命中该 appKey 下全部生图模型配置；选择时只命中指定生图模型 |

### 11.2 更新规则

- 批量更新只覆盖命中记录的 `quota`。
- 覆盖方式为整包覆盖，不做字段级合并。
- 抽屉内需明确提示用户确认 appKey 和生图模型条件无误。原型采用抽屉顶部警示文案即可满足提示要求，不强制增加二次确认弹窗；如前端项目对批量高风险操作已有统一二次确认规范，可按项目规范补充。
- 提交成功后关闭抽屉并刷新列表。
- 若未命中任何记录，应给出“未命中任何记录”提示。

## 12. quota 规则

quota 支持两种填写方式：

1. 结构化填写：逐行填写 key 和额度值。
2. JSON 填写：直接输入完整 JSON 对象。

两种方式提交前统一校验，规则如下：

| 规则项 | 要求 |
| --- | --- |
| 数据类型 | 必须为 JSON 对象 |
| 空对象 | 不允许 |
| key 格式 | 仅允许字母、数字、下划线，且必须以字母或下划线开头 |
| key 重复 | 不允许 |
| value 格式 | 必须为整数 |
| value 范围 | 0-20 |

示例：

```json
{
  "total": 20,
  "scene": 3,
  "closeUp": 2
}
```

本期前端先按上述规则做提交前校验，其中 `value` 范围按 `0-20` 执行。若已上线接口对 `quota` 的字段集合或数值范围有更细的限制，前端应展示接口返回的错误信息并配合联调；若接口规则与本需求冲突，需联系后端和产品确认最终口径。

## 13. 交互与体验要求

- 抽屉和删除弹窗支持点击关闭按钮关闭。
- 抽屉和删除弹窗支持按 Esc 关闭。
- 抽屉打开时应聚焦到关闭按钮或抽屉内首个可操作元素，关闭后尽量恢复到触发入口。
- 筛选标签展示用户输入内容时必须做转义，避免特殊字符影响页面。
- 表格无数据时展示空状态提示。
- 列表中 quota 可截断展示，但查看详情中必须展示完整 JSON。
- 美工管理员配置和运营管理员配置切换时颜色保持一致，统一使用蓝色主题。

## 14. 后端接口与联调说明

本需求不要求新增后端接口，前端直接接入已上线接口能力。前端实现时需要完成：

- 按页面类型区分美工管理员配置和运营管理员配置的数据请求。
- 接入列表查询、详情、新增、编辑、删除、批量更新接口。
- 按接口实际字段完成前端字段映射。
- 按接口返回处理成功、失败、空结果、权限不足等状态。

接口状态处理要求：

- 查询列表、提交表单、删除、批量更新时应有加载态或按钮禁用态，避免重复提交。
- 接口成功后按当前操作刷新列表：新增成功后回到第一页；编辑、删除、批量更新成功后刷新当前查询条件下的列表。
- 删除导致当前页无数据时，可按后台通用分页策略回退到上一页或展示空状态。
- 接口失败时展示错误提示；若接口返回明确错误文案，优先展示接口文案。
- 权限不足时展示无权限提示或跳转无权限页面，按现有后台规范执行。
- 列表无数据时展示空状态，不应误提示为接口失败。

若出现以下情况，前端可联系后端联调支持：

- 接口缺少页面所需字段。
- 接口字段含义与页面展示不一致。
- quota 校验规则与页面规则不一致。
- 生图模型枚举与页面展示不一致。
- 权限控制与指定管理员范围不一致。
- 批量更新命中范围或覆盖规则与需求不一致。

## 15. 验收重点

| 模块 | 验收点 |
| --- | --- |
| 页面入口 | 两个独立页面/路由可访问，命名分别为美工管理员配置、运营管理员配置 |
| 权限 | 非指定管理员不可进入或操作配置页面 |
| 查询 | 各筛选条件可正常查询，切换页面时清空筛选状态 |
| 列表 | 字段展示完整，排序默认按更新时间倒排，分页和重置行为符合要求 |
| 新增 | 必填、唯一性、quota 校验生效，成功后刷新列表 |
| 查看 | 可完整查看基础信息和 quota JSON |
| 编辑 | 用户ID、生图模型、用户名只读；appKey、quota、启用状态可更新 |
| 删除 | 二次确认后删除，成功后刷新列表 |
| 批量更新 | 按 appKey 和可选生图模型命中，quota 整包覆盖，有明确风险提示 |
| quota | 结构化填写和 JSON 填写规则一致，错误提示清晰 |
| 接口状态 | 加载、成功、失败、无权限、空结果、防重复提交均有明确处理 |
| 交互 | 抽屉/弹窗支持关闭按钮和 Esc 关闭，无明显控制台错误 |

## 16. 开放问题

- 指定管理员名单和权限配置方式需以前端所在后台的权限体系为准。
- 生图模型枚举是否会继续扩展，需要以后端接口或统一枚举源为准。
- 如后续确认不同生图模型的 quota value 范围不同，需补充模型维度的校验规则；本期先统一按 `0-20` 执行。
